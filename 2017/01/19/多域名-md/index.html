绑定域名三步走

  1. 在博客sourse目录CNAME文件中添加域名记录,可以写多个,添加多个时直接在新行添加记录就好了
  2. 为自己想要绑定的域名添加云解析DNS; 设置记录类型为CNAME,主机记录看需求,可配合自定义域名随意设置;@是域名前没有值
  3. 在托管平台pages服务里设置自定义域名
  4. 现在可以访问本站的域名(都不用加www,当然设置www的记录类型很简单,只是自定义域名个数比较紧缺;也不用加协议,浏览器会帮忙0.0)

其他

  1. 网易云万网上CNAME上的主机记录是用来解析域名之前的字符串的,值为@时表示域名之前不加任何字符串,值为www表示域名之前有www,值为其他字符串时表示,www.字符串.域名

  2. github 和 coding 都用的是git方式管理代码, 和ssh安全验证(C:\Users\高洋.ssh\id_rsa)SSH需要具体了解

  3. 在网站的配置文件中设置代码部署deploy时,如果是 git@git.coding.net:用户名/项目名.git则会部署到具体项目上(…);

  4. 当一个站上上只有一个项目时将项目名写成用户名就可以在访问时省略项目名,用用户名直接访问

window下不能直接创建一个点开头的文件,eg: .txt 不能被创建

  • 想要创建,只要在后缀的后面再接一个点就可以了,eg:.txt.
    • 思考:测试发现在文件后面无论加几个点,在确定后都会消失;
    • 应该是windows在判断文件名时如果只有一个点那么,点之前必须有字符串作为文件名
      eg: 123. 保存后的文件名为123
    • 如果有多个点,那么最后一个后面跟有字符串的点会被当做后缀名来使用;
      eg: 1.2….3.4.5….. 保存后的文件名为1.2….3.4.5
    • 如果文件后面有不跟字符串的点将会被忽略
      eg: 1.txt…..
